<task type="auto">
  <name>Task 2: Make linkToDizionario filter language-aware</name>
  <files>scripts/templates/helpers.js, eleventy.config.mjs</files>
  <action>
**Problem:** The current `linkToDizionario` filter in helpers.js generates links like `dizionario.html#anchor`. On IT pages this is correct (relative link). But on EN/FR pages this would link to `/en/dizionario.html` or `/fr/dizionario.html` (which may not exist). All dizionario links should point to the IT dizionario page.

**Fix:** Make the filter accept a language parameter. For IT pages, use relative `dizionario.html#anchor`. For EN pages, use `/dizionario.html#anchor` (absolute to root). For FR pages, also use `/dizionario.html#anchor`.

Update `helpers.js` `linkToDizionario` function to accept a second parameter `lang`:

```js
function linkToDizionario(documentName, lang) {
  // ... existing matching logic ...

  // Determine dizionario URL prefix
  let dizionarioBase = 'dizionario.html';
  if (lang === 'en' || lang === 'fr') {
    // Link to IT dizionario (absolute path from root)
    dizionarioBase = '/dizionario.html';
  }

  // In the replacement loop, change:
  const link = `<a href="${dizionarioBase}#${r.anchorId}" class="doc-link">${escapeHtml(r.original)}</a>`;
  // ... rest unchanged
}
```

**Update filter registration in eleventy.config.mjs:**

The current filter is registered as:
```js
eleventyConfig.addFilter("linkToDizionario", helpers.linkToDizionario);
```

Liquid filters receive the input value as the first argument and additional arguments as subsequent params. In templates, the filter is called as `{{ documentName | linkToDizionario }}`. To pass language, update the templates to use `{{ documentName | linkToDizionario: lang }}` where `lang` is the page's language front matter variable.

However, the simpler approach is to check if the filter is called from Liquid with an optional second parameter. In 11ty, Liquid filters receive extra arguments after the value. So register as-is — the filter already accepts `(documentName, lang)`.

**Update Liquid templates** that use the filter. Find all usages of `| linkToDizionario` in:
- `src/pages/documents-primo.liquid`
- `src/pages/documents-rinnovo.liquid`
- `en/src/pages/documents-primo-en.liquid`
- `en/src/pages/documents-rinnovo-en.liquid`
- `fr/src/pages/documents-primo-fr.liquid`
- `fr/src/pages/documents-rinnovo-fr.liquid`

Change from:
```liquid
{{ docName | linkToDizionario | safe }}
```
To:
```liquid
{{ docName | linkToDizionario: lang | safe }}
```

This passes the page's `lang` variable (set in front matter: `it`, `en`, or `fr`) to the filter, which uses it to determine the correct dizionario URL prefix.

**Backward compatibility:** If `lang` is undefined (shouldn't happen but safety), default to `'it'` behavior (relative path).
