## v1.4 Requirements

### Error Reporting

- [x] **ERR-01**: "Segnala errore" button visible on all content pages (permit pages, document pages, guide pages)
- [x] **ERR-02**: Button positioned top-right of page content, opposite breadcrumbs
- [x] **ERR-03**: Button redirects to Typeform with current page URL as parameter
- [x] **ERR-04**: Typeform captures: error type dropdown + description field (User Setup)
- [x] **ERR-05**: Error type options: Informazione errata, Informazione mancante, Info obsoleta, Link rotto, Problema di visualizzazione (User Setup)

### Dropdown Navigation

- [x] **NAV-01**: Database menu item has dropdown with 2 items: "Database di permessi", "Che documenti porto in Questura"
- [x] **NAV-02**: Guide menu item has dropdown with 3 items: "Protezione internazionale", "Ricongiungimento familiare", "Dizionario"
- [x] **NAV-03**: Test menu item has dropdown with 2 items: "Posso AVERE un permesso?", "Posso RINNOVARE il permesso?"
- [x] **NAV-04**: Desktop: dropdowns open on hover
- [x] **NAV-05**: Mobile: flat list in hamburger menu (no dropdowns)
- [x] **NAV-06**: Dropdown styling consistent with existing header design (white background, teal text)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Inline error form | User prefers redirect to Typeform for consistency |
| Contact form changes | Existing Typeform contact works fine |
| Nested mega-menu dropdowns | Simple dropdowns sufficient for current content |
| Mobile dropdown accordions | User prefers flat list on mobile |
