# Roadmap: v2.0 Multilingual Support (English)

**Status:** Backburner (Future Milestone)
**Priority:** High — crucial for reaching non-Italian speaking users
**Estimated Phases:** 4
**POC Completed:** 2026-01-31

---

## Milestone Goal

Enable English-speaking users to access the full site with high-quality AI translations validated by human reviewers through a user-friendly interface.

---

## Context

### POC Completed (2026-01-31)

- **4 sample pages translated** (database, permesso-studio, permesso-richiesta-asilo, documenti-studio-primo)

## Phases

### Phase 19: Translator Review Interface

**Goal:** Non-technical translators can review and correct AI translations through a web interface

**Deliverables:**
- Web-based review dashboard (HTML/JS, no backend required)
- Side-by-side IT/EN view with inline editing
- Progress tracking per page (pending → reviewed → approved)
- Export corrected HTML files
- Shareable via GitHub Pages or static hosting

**Success Criteria:**
1. Translator can load any IT/EN page pair
2. Translator can edit EN text inline
3. Changes are saved locally (localStorage) and exportable
4. Dashboard shows review progress across all pages

---

### Phase 20: Batch Translation Pipeline

**Goal:** All 208 pages translated using AI with glossary enforcement

**Deliverables:**
- Node.js script to batch translate pages
- Glossary term replacement post-processing
- Structural preservation (HTML tags, links, attributes)
- Automatic path updates (IT links → EN links)
- Translation manifest tracking completed pages

**Success Criteria:**
1. Running `npm run translate:en` generates all EN pages
2. All pages pass verification script with no errors
3. Internal links point to `/en/` versions
4. `lang="en"` attribute set on all pages

---

### Phase 21: Human Review & Corrections

**Goal:** All translations reviewed and corrected by human translators

**Deliverables:**
- Prioritized review queue (high-traffic pages first)
- Corrections applied via review interface
- Final verification pass
- Sign-off checklist per page category

**Success Criteria:**
1. 100% of permit pages reviewed
2. 100% of document pages reviewed
3. All glossary terms used consistently
4. Legal/technical accuracy verified

---

### Phase 22: Language Switching Integration

**Goal:** Users can switch between IT and EN versions seamlessly

**Deliverables:**
- Language switcher points to correct EN pages
- EN pages link back to IT equivalents
- hreflang tags for SEO
- EN sitemap.xml
- 404 fallback handling

**Success Criteria:**
1. Clicking EN flag on any IT page goes to EN equivalent
2. Clicking IT flag on any EN page goes to IT equivalent
3. Google can discover EN pages via sitemap
4. Missing EN pages show graceful fallback
